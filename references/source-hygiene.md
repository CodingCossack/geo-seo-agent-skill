# Source and claim hygiene

Do not repeat hard-coded market claims just because they appear in old repo
materials.

Treat the following as untrusted unless freshly verified for the current run:

- GEO market-size forecasts
- AI traffic conversion multipliers
- exact overlap percentages between platforms
- exact word-count claims for citation likelihood
- exact correlations between brand mentions and AI visibility

## Allowed uses of heuristics

You may use heuristic language such as:

- likely
- directional
- usually helps
- common pattern
- often easier to quote

Do not convert those heuristics into fake statistics.

## Evidence hierarchy

1. observed site evidence from the current target
2. deterministic script output from this bundle
3. current external documentation or current platform docs when verified live
4. clearly-labeled heuristics

## Reporting rule

Every major claim in an audit should be one of these:

- directly observed on the target site
- inferred from observed evidence and labeled as inference
- externally verified for the current run

Anything else gets removed.
